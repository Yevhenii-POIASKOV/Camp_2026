# E–commerce ETL Pipeline

## Архітектура проєкту

Проєкт розділений на два основні етапи:

1. **Python App**: Очищує дані та завантажує в базу.
2. **PostgreSQL**: Реляційна база даних для зберігання таблиць та аналітики.

### Структура папок

* `src/`: Вихідний код очищення (`cleaner.py`) та бази даних (`database.py`).
* `data/`: Вхідні сирі CSV–файли (`customers.csv`, `products.csv`, `orders.csv`, `order_items.csv`).
* `output/`: Результати очищення у форматі CSV та файл бази даних.
* `logs/`: Автоматично згенеровані логи виконання пайплайну.
* `main.py`: Головний файл для запуску всього процесу.

## Запуск

Для запуску проєкту вам потрібен лише **Docker** та **Docker Compose**.

1. Розмістіть ваші вхідні CSV–файли в папку `data/`.
2. Запустіть команду в терміналі:

    ```bash
    docker–compose up ––build
    ```

Ця команда автоматично збере образ Python, встановить необхідні бібліотеки (`pandas`, `sqlalchemy`, `psycopg2–binary`), запустить базу даних та виконає весь цикл обробки даних.

## Аналітика

Після завантаження даних у PostgreSQL автоматично створюються 6 аналітичних таблиць для бізнес–аналізу:

1. **`analytics_category_revenue`**: Виручка та кількість замовлень у розрізі категорій товарів.
2. **`analytics_customer_value`**: Рейтинг клієнтів за загальною сумою витрат (LTV).
3. **`analytics_sales_by_country`**: Географічний розподіл продажів та виручки.
4. **`analytics_top_products`**: Топ найбільш популярних товарів за кількістю проданих одиниць.
5. **`analytics_monthly_trends`**: Динаміка кількості замовлень за місяцями.
6. **`analytics_order_metrics`**: Розрахунок загальної вартості та кількості товарів для кожного окремого замовлення.

## Логування та звітність

* Пайплайн генерує детальні логи в папку `logs/` із міткою часу .
* Після виконання в консоль виводиться підсумкова таблиця **STATISTICS**, яка показує кількість видалених рядків для кожної таблиці та причину видалення.
